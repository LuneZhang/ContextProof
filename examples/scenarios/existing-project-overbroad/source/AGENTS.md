## Working Rules

- Always follow best practices and write clean code.
- Before every task, read the entire repository and inspect all files so you do
  not miss anything.
- Always run all tests before responding, even for documentation-only changes.
- Use your judgment when deciding whether a change is good enough.

## Project Notes

- Main application code lives in `src/app`.
- Shared test helpers live in `tests/helpers.py`.
- Run `pytest tests/unit` before submitting code changes.
